# Production Checklist

**Covers:** a pre-ship gate for correctness, UX, performance, maintainability, and observability.

**Read when:** wrapping up a flow feature, reviewing a PR, or before deploying.

## Correctness

- [ ] Package is `@xyflow/react`; no stray `reactflow` imports.
- [ ] `@xyflow/react/dist/style.css` imported once, app-wide (Tailwind v4: after Tailwind in the global stylesheet).
- [ ] `<ReactFlow>` parent has explicit, non-zero width **and** height in every layout it renders in.
- [ ] Flow is controlled: `onNodesChange`/`onEdgesChange`/`onConnect` wired; changes applied immutably.
- [ ] All node/edge IDs are unique, string, and stable; generated by an owned source (uuid/nanoid/counter).
- [ ] `nodeTypes`/`edgeTypes` defined at module scope; every `type` used exists in the registry.
- [ ] Node/edge `data` is typed (discriminated union); no `any`.
- [ ] If persisted: `data` is JSON-serializable; graph schema is versioned; serialize↔deserialize round-trips (tested).
- [ ] Dynamic handles call `updateNodeInternals`; connection rules enforced in `isValidConnection`.
- [ ] Hooks run inside `<ReactFlowProvider>`/`<ReactFlow>`.

## UX & accessibility

- [ ] Interactive node content has `nodrag`; scrollable content `nowheel`; overlays `nopan`.
- [ ] Selection/hover/focus affordances present and not color-only; focus outlines preserved.
- [ ] Handles are comfortably clickable; connection feedback (valid/invalid cursor) works.
- [ ] `colorMode` / dark mode renders correctly; contrast acceptable in both modes.
- [ ] Essential labels legible at default zoom; `<Controls>`/`<MiniMap>` present where useful.
- [ ] Keyboard interaction works (focus, move, delete).

## Performance

- [ ] Custom node/edge components are `React.memo` or module-scope.
- [ ] Prop functions `useCallback`'d; object/array props (`defaultEdgeOptions`, `snapGrid`) `useMemo`'d.
- [ ] No inline object/array literals on `<ReactFlow>` or hot inner elements.
- [ ] Store subscriptions are narrow (no whole-`nodes`-array subscriptions in many components).
- [ ] Single-node updates use `updateNodeData`, not whole-array rebuilds.
- [ ] Large graphs: `hidden` collapsing and/or `onlyRenderVisibleElements` evaluated; node visuals lightweight; few/no animated edges at scale.
- [ ] Verified at the real target node/edge counts.

## Maintainability

- [ ] Layered layout: presentational nodes/edges, single registry, pure `lib/flow/` logic, isolated state.
- [ ] Domain logic is out of render components.
- [ ] Layout logic is a pure function, not entangled with rendering.
- [ ] Node creation goes through factories; types live in `types/`.
- [ ] File names kebab-case; type strings consistent and match registry keys.

## Observability & robustness

- [ ] Errors in tool/data loading surfaced to the user (no silent blank canvas).
- [ ] SSR/SSG flows are client-bounded.
- [ ] Tests cover factories, transforms, connection rules, and (if persisted) round-trip.
- [ ] No console warnings ("node type not found", duplicate keys, provider missing) in normal use.
- [ ] If using React Flow devtools/debugging, it's gated to dev builds.

## Agent instructions

1. Run this list before reporting a flow feature done; report any unchecked item explicitly rather than glossing over it.
2. For each failed item, route to the owning module and fix or flag.
